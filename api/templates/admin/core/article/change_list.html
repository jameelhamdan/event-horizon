Each button force-dispatches one stage from the registry (fetch / analyze / annotate / aggregate — using the stage's own selection and chunk size, same as the dashboard's per-stage Reprocess buttons). Analyze is the full cloud-LLM pass for freshly-fetched live articles only (needs LIVE_LLM_ENABLED on) — historical/backfill articles always go through the free on-prem Annotate pass instead. Run full pipeline → Events dispatches every enabled stage in pipeline order in one tick (fetch → analyze/annotate → refine → aggregate → tag → route) — each stage still runs as its own queued job, not literally chained, so Events may take a few worker cycles to appear.
